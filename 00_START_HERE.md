﻿﻿# Пакет Для Параллельной Разработки (Код-Код)

Эта папка собрана как переносимый контекст проекта для параллельной разработки в другом чате/агенте.

## Что внутри

1. `research/object_presentation_tz/`  
   Полный набор ТЗ, правил, решений и отчетов по gate-проходу.

2. `tools/normalize_docs_encoding.ps1`  
   Скрипт авто-нормализации кодировки документов (UTF-8 BOM) перед чтением.

3. `CLAUDE_COWORK_PROMPT.md`
   Готовый стартовый промпт для параллельного запуска в Claude/Cowork.
4. `tools/setup_utf8_windows.ps1`
   One-time настройка UTF-8 baseline для PowerShell/Git/переменных окружения пользователя.

## Главный источник требований

- Сначала читать: `research/object_presentation_tz/MASTER_SPEC_v2.md`
- Затем: `research/object_presentation_tz/00_README_START_HERE.md`
- При конфликте документов: приоритет у `MASTER_SPEC_v2.md`

## Зафиксированные ключевые решения

- Финальный формат слайда: **A4 альбомный (29.7 x 21.0 см)**.
- Язык интерфейса и подписи: русский.
- Слайд 2: блок "Преимущества" обязателен.
- Разрешено использовать готовые внешние модули (npm/GitHub), если это ускоряет разработку и не ухудшает качество/соответствие ТЗ.
- Пакет для Claude/Cowork сделан как **spec-only**: реализацию делать с нуля по ТЗ, не копируя текущий код.

## Как использовать пакет в другом контуре

1. Передать всю папку `код-код` целиком.
2. Один раз выполнить baseline-настройку UTF-8:
   `powershell -ExecutionPolicy Bypass -File tools/setup_utf8_windows.ps1`
3. Перезапустить терминал/вкладки редактора.
4. Запустить нормализацию кодировки:
   `powershell -ExecutionPolicy Bypass -File tools/normalize_docs_encoding.ps1 -Root .`
5. Попросить начать с чтения `00_START_HERE.md`.
6. Попросить строго следовать `MASTER_SPEC_v2.md`.
7. Попросить делать реализацию с нуля и запускать цикл проверки PDF до совпадения с образцом.
