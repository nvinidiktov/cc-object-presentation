﻿﻿# Best-Practice Questions (Final Pre-Implementation Checklist)

Use this checklist once before coding starts:

1. What is the exact acceptance definition for a "good" PDF output?
2. Which fields are strict blockers for export vs warnings only?
3. What maximum project size is expected in real use (photos/slides/text)?
4. What is acceptable generation/export time on your actual laptop?
5. What should happen if AI output conflicts with strict layout rules?
6. Is every user action recoverable (undo/redo/version restore)?
7. How do we prevent accidental destructive actions in media management?
8. How is fallback behavior communicated to user without confusion?
9. Which logs are mandatory to debug failures after overnight runs?
10. What exactly is "done" for MVP vs phase-2 backlog?
11. Which visual quality checks are must-have before calling design complete?
12. Which file naming and storage conventions are non-negotiable?
13. What is the smallest smoke-test set required before each gate handoff?
14. What are the top 3 risks that can kill delivery timeline?
15. Who approves each gate if both Codex and Claude outputs differ?
